# 跨领域数据挖掘竞赛 --从数据到决策的全流程实战
| 维度 | 说明 |
| --- | --- |
| 团队规模 | 4 人 / 组(固定) |
| 项目周期 | 5 周 |
| 难度级别 | 中高(相当于小型科研课题或 Kaggle 竞赛) |
| 核心要求 | 每个组员有明确的技术分工,项目需包含多个数据源融合､多种算法系统性对比､ 可交互的成果展示 |

## 一、项目选项(三选一)
### 选项A:多源数据融合挖掘
整合至少2 个不同来源/模态的数据集进行分析。
示例选题:
- 房价预测:结构化数据(房屋特征,+ 非结构化数据(房源描述文本/图片)
- 信用风险评估:用户基本信息 + 交易流水时序 + 社交媒体行为
- 城市犯罪预测:历史犯罪记录 + 人口统计数据 + POI 数据 + 天气数据

### 选项B:算法系统性对比研究
针对同一任务,实现并对比至少5 种不同类型的算法,进行深入的理论和实验分析。
示例选题:
- 异常检测算法全面对比:统计方法､隔离森林､LOF､One-Class SVM､自编码器
- 时间序列预测方法对比:ARIMA､Prophet､LSTM､Transformer､XGBoost(特征工程 版)

### 选项C:端到端应用系统开发
构建一个完整的数据挖掘应用,包含数据处理､模型训练､评估和轻量级交互界面。
示例选题:
- 电影推荐系统:协同过滤 + 基于内容的推荐 + 冷启动处理 + Streamlit 演示界面
- 股票情绪分析:新闻/社交媒体爬取 + NLP 情感分析 + 股价关联分析 + Dashboard

## 二、团队角色与分工
每位组员承担一个明确的技术角色,在报告中用贡献表说明,下表可供参考

| 角色 | 职责 | 可考核的输出 |
| --- | --- | --- |
| 数据工程负责人 | 数据获取､清洗､融合､特征工程､ 数据管道构建 | 数据预处理代码､特征文档､数据 血缘图 |
| 算法负责人(建模) | 核心算法实现 / 调优､对比实验设 计､超参数优化 | 模型训练代码､实验记录表､模型 对比分析 |
| 评估与分析负责人 | 评估指标设计､统计显著性检验､ 结果可视化､错误分析 | 评估代码､可视化图表､分析报告 章节 |
| 工程与交付负责人 | 代码整合､ Pipeline 构建､交互界 面 / 报告生成､版本管理 | 可运行完整代码､ README ､演 示视频 /PPT |

额外要求:
* 每个角色的贡献在报告中要有可量化的证据(如:某段代码的commit 记录､某章节的作者署名)
* 鼓励跨角色协作,但主责分明

## 三、项目交付物
| 序号 | 交付物 | 要求 | 团队分值占比 |
| --- | --- | --- | --- |
| 1 | 项目计划书 | 选题背景､数据来源､分工计划､时 间表 (第 13 周前提交) | 5% |
| 2 | 代码仓库 | GitHub/Gitee 仓库,完整 commit 历 史,每个组员有贡献记录 | 10% |
| 3 | 完整代码 | 模块化､有注释､可一键运行主流程 | 25% |
| 4 | 项目报告 | 10-20 页,学术论文格式,包含摘要､ 引言､方法､实验､讨论､结论 | 30% |
| 5 | 团队分工声明 | 逐条说明每个人完成的具体代码文 件､报告章节､工作量百分比 | 5% |
| 6 | 答辩 PPT + 演示视频(可选) | 10 分钟(讲解 + 系统演示 / 结果展示) | 25% |
| 7 | 项目海报(加分项) | A0/A1 尺寸,适合学术展示日 | |

## Tips
这个项目的设计目标是模拟真实的数据科学团队工作｡在业界,你不会一个人完成所有事,但你 需要:
- 在自己的领域成为专家
- 与队友高效协作
- 对最终结果负责
