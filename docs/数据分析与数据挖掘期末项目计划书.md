# 一、选题背景与意义

皮肤癌是全球最常见的癌症之一，早期诊断对患者预后有决定性影响。然而皮肤病变的肉眼识别高度依赖医生经验，存在漏诊率高、基层医疗资源匮乏等问题。借助数据挖掘与机器学习技术，可以构建辅助诊断系统，提升诊断效率与准确性。

本项目基于公开的 HAM10000 数据集（Human Against Machine with 10000 training images），该数据集由维也纳医科大学与澳大利亚昆士兰皮肤科诊所联合收集，包含：

- 10,015 张皮肤镜图像，覆盖 7 种常见皮肤病变类型

- 配套结构化元数据：患者年龄、性别、病灶部位、诊断确认方式

- 标注由皮肤科专家完成，超过50%经病理切片确认

本项目将构建一个完整的端到端数据挖掘应用，从数据预处理、模型训练到可交互演示界面，实现对7类皮肤病变的自动分类诊断。

# 三、技术方案概述

## 3\.1 整体架构

系统采用端到端流水线设计，分为以下四个模块：

- 数据处理模块：图像预处理、元数据清洗、类别不平衡处理

- 数据挖掘模块：进行数据挖掘分析，可视化类别、年龄、性别、病灶部位分布，检查异常值、缺失值

- 建模模块：多模型训练（Baseline CNN \+ EfficientNet\-B0迁移学习），结合元数据特征的融合分类器；支持模型对比与消融实验

- 评估模块：多维度指标计算（Macro F1、AUC、混淆矩阵）与消融实验对比

- 交互界面模块：基于Streamlit的Web演示系统

## 3\.2 核心技术选型

|**模块**|**技术栈**|**说明**|
|---|---|---|
|数据处理|Python / Pandas / Albumentations|图像增强、缺失值填补、过采样（SMOTE）|
|模型训练|PyTorch / timm|EfficientNet\-B0迁移学习，融合元数据特征; Baseline CNN; 支持多模型对比|
|实验对比|Scikit\-learn|纯图像模型 vs 纯元数据模型 vs 融合模型消融实验|
|可视化评估|Matplotlib / Seaborn|混淆矩阵、ROC曲线、类别分布图|
|交互界面|Streamlit|图像上传、元数据输入、预测结果展示|
|版本管理|Git / GitHub|每人独立分支，PR合并，保留commit记录|

## 3\.3 系统功能规划

交互系统包含以下核心功能页面：

- 诊断页面：上传皮肤镜图片 \+ 输入患者元数据（年龄/性别/部位），输出病变类别预测与各类置信度

- 模型评估页面：展示测试集上的混淆矩阵、Macro F1、各类别AUC

- 消融实验页面：对比纯图像模型、纯元数据模型、融合模型的性能差异

- 数据探索页面：HAM10000类别分布、样本示例、年龄/性别分布可视化
