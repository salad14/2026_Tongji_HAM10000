> 本文档记录项目的技术决策依据、实验设计逻辑和执行计划，供团队成员参考。

## 一、选题定位

**课程**：数据挖掘与分析
**选项**：C — 端到端应用系统开发
**数据集**：HAM10000（皮肤癌 MNIST）
**核心主张**：通过多模态数据融合（图像 + 结构化元数据）提升皮肤病变分类精度，并以可交互系统呈现完整数据挖掘流程。

### 为什么选这个数据集

- 图像（皮肤镜）+ 结构化元数据（年龄/性别/部位）天然构成多模态场景
- 存在严重类别不平衡，提供真实的数据挖掘挑战（非玩具问题）
- 医学背景使项目有实际应用意义，答辩演示效果直观
- 数据集公开、有完整标注，不需要爬取或手动标注

## 二、技术方案决策

### 2.1 为什么选 EfficientNet-B0

| 选项 | 优点 | 缺点 | 决策 |
|------|------|------|------|
| 从零训练小 CNN | 完全自定义 | 数据量不足，效果差 | ❌ |
| ResNet-50 | 成熟稳定 | 参数量较大，Colab 显存压力 | 备选 |
| **EfficientNet-B0** | 参数少、精度高、适合迁移学习 | — | ✅ 主选 |
| ViT / Swin | 最新架构 | 对数据量要求高，调参复杂 | ❌ |

### 2.2 元数据融合策略

选择**特征拼接（Late Fusion）**：

```
image_feat (1280-d) + meta_feat (64-d) → concat → FC(1344, 256) → FC(256, 7)
```

理由：
- 实现简单，代码改动量小
- 两个分支可以独立训练和验证，便于消融实验
- 相比早期融合（Early Fusion），避免不同模态特征空间不匹配的问题

### 2.3 类别不平衡处理策略

采用**类别权重**方案（而非过采样）：

```python
class_weights = 1.0 / class_counts
class_weights = class_weights / class_weights.sum()
criterion = CrossEntropyLoss(weight=class_weights)
```

理由：
- 过采样（SMOTE）对图像数据效果有限（SMOTE 基于特征空间插值，图像插值语义不明确）
- 图像增强（Albumentations）对少数类样本做随机变换，间接达到数据增强效果
- 类别权重实现简单，可解释性强

## 三、消融实验设计

### 目的

证明元数据的加入对分类性能有贡献（即使提升有限，实验本身即为研究贡献）。

### 三组实验

| 实验名 | 输入 | 模型结构 |
|--------|------|---------|
| `image_only` | 224×224 图像 | EfficientNet-B0 → FC(1280, 7) |
| `meta_only` | [age, sex, loc] | MLP(3→32→64→7) |
| `fusion` | 图像 + 元数据 | EfficientNet-B0 + MLP → Concat → FC(1344→256→7) |

### 评估指标

- **主指标**：Macro F1（所有类别权重相等，不受多数类影响）
- **辅助**：各类别 AUC（One-vs-Rest）、混淆矩阵
- **不使用** Accuracy（在严重不平衡场景下无意义）

### 预期结论方向

```
meta_only << image_only < fusion
```

若 fusion 未显著超过 image_only，报告中应分析原因（元数据字段有限、编码方式待优化等），这本身是有价值的发现。

## 四、Streamlit 系统功能规划

### 页面 1：诊断（核心功能）

- 上传皮肤镜图片
- 输入患者元数据（年龄滑块、性别单选、部位下拉）
- 输出：预测病变类别 + 7类置信度条形图 + 病变说明文字

### 页面 2：模型评估

- 展示测试集上三组实验的 Macro F1 对比表格
- 融合模型的混淆矩阵热力图
- 各类别 ROC 曲线（One-vs-Rest）

### 页面 3：消融实验对比

- 上传同一张图，三个模型同时推理，对比输出结果
- 用于答辩现场直观展示多模态融合的价值

### 页面 4：数据探索（EDA）

- 类别分布柱状图（含样本数和占比）
- 年龄分布直方图（按类别分组）
- 性别 / 部位分布饼图
- 各类别随机样本展示（3×7 图像网格）
